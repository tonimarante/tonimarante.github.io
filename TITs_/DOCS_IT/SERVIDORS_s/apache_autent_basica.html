Instead of playing around with the original httpd.conf, copy it to a new httpd.conf.debug and use this new httpd.conf.debug file with Apache for testing purpose as shown below using option -f.
 

  1. apachectl -f conf/httpd.conf.debug

 
 

  1. httpd -k start -f conf/httpd.conf.debug

Use a temporary DocumentRoot without modifying httpd.conf

  1. httpd -k start -c "DocumentRoot /var/www/html_debug/"

 
 
 
If you want to go back to original configuration using the default DocumentRoot (/var/www/html), simply restart the Apache as shown below.
 

  1. httpd -k stop
  2. apachectl start

Display the modules compiled inside Apache using option
-l
 

  1. httpd -l

Compiled in modules:
When you pass option -l, to httpd, it will display only the static modules. Passing option -M, will display both static and shared modules as shown below.
 

  1. httpd -M

Use option -t to validate whether there are any issues with a specific Apache configuration file

  1. httpd -t -f conf/httpd.conf.debug

Apache, autent. bàsica

Per ficar un password d'entrada a un lloc web:

-creem un usuari i un password amb el commandament htpasswd, p.e. htpasswd
/etc/mariseg progra. Això creara una base de dades d'usuaris a l'arxiu
(/etc/mariseg) i el primer usuari introduït serà progra (ens preguntarà el
password per aquest usuari).

-Si l'arrel de la plana web està a /home/bmw/public_html, afegirem aquest codi a
l'arxiu httpd.conf:

<Directory /home/bmw/public_html>
AuthName "Programari"
AuthType Basic
AuthUserFile /etc/mariseg
Require user progra

#Opcions indexat
Options Indexes Multiviews
IndexOptions FancyIndexing FoldersFirst SuppressDescription

</Directory>

Amb això només podrà entrar l'usuari progra, encara que tinguem creats més.
